# Kytsunia

Telegram бот з характером для приватних чатів.

## Розробка

Потрібен Node 22+ і pnpm.

```bash
pnpm install
cp .env.example .env
# заповни .env
pnpm dev
```

## Vision (фото)

Кицюня вміє бачити фото. Працює коли модель — Sonnet (або інша з vision).

Підтягуються тільки явно адресовані фото — три канали в порядку пріоритету:

1. **Тригер** — фото (або альбом) у самому повідомленні до Кицюні.
2. **Reply-chain** — якщо тегнули в reply, Кицюня йде назад по ланцюгу
   `reply_to → reply_to → ...` (включно через свої власні відповіді) і
   знаходить найближче фото в треді. Так Кицюня «памʼятає» картинку поки
   ти продовжуєш про неї питати.
3. **TTL fallback** — якщо нема ні тригера, ні reply, але хтось у чаті
   щойно (за останні ~2 хв) кинув фото — Кицюня подивиться на нього.

«Стара» історія чату фото в контекст **не несе** — інакше модель фіксується
на них при кожному текстовому реплаї.

Власні відповіді Кицюні зберігаються в `messages` (з `sender_id` = bot id) —
це потрібно для роботи reply-chain і покращує контекст у звичайних розмовах.

Налаштування (env):

- `KYTSUNIA_VISION_ENABLED` (default `true`) — фіче-флаг
- `KYTSUNIA_MAX_PHOTOS_TOTAL` (8) — cap на кількість фото в одному запиті
- `KYTSUNIA_MAX_PHOTOS_PER_ALBUM` (5) — cap на один альбом
- `KYTSUNIA_VISION_ALBUM_DEBOUNCE_MS` (1500) — затримка для збору сіблінгів альбому
- `KYTSUNIA_VISION_THREAD_DEPTH` (5) — макс. кроків у reply-chain
- `KYTSUNIA_VISION_TTL_MS` (120000) — вікно TTL fallback; `0` вимикає

Скачані фото кешуються в SQLite за `file_unique_id` — повторні відповіді в
тому ж треді не качають Telegram заново.

## Privacy

`/забудь мене` (opt-out) стосується **тільки персональних профілів** —
Кицюня перестає вивчати твій стиль. Повідомлення в контексті чату
(включно з фото) залишаються видимими для бота, бо без них він не зможе
підтримати розмову.
